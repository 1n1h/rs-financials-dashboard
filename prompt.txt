Build a React web app financial dashboard for a bookkeeping client named Rosemary Schafer.
This will be deployed on Vercel via GitHub. Use React (Vite), Tailwind CSS, and 
react-chartjs-2. Style inspiration: fathom.video — dark background, bright cyan accent, 
glassmorphism cards, premium SaaS feel.

---

TECH STACK:
- React (Vite)
- Tailwind CSS
- react-chartjs-2 + Chart.js
- SheetJS (xlsx) — to parse the Excel file directly
- PapaParse — for CSV uploads
- OpenRouter API — AI chat assistant (server-side proxied)
- Vercel Edge Functions — to proxy OpenRouter API key securely
- React Router

EXCEL DATA SOURCE:
The file is: 2025_Rose_NEW_and_IMPROVED_Financials.xlsx
It is located in the /data folder of this project.

This file has 4 sheets:
  1. "RS P&L - 2025"       ← PRIMARY: use this for all income, expense, and net data
  2. "RS BS - 2025"        ← use for account balances by month
  3. "RS Personal P&L - 2024"  ← use for year-over-year comparisons (optional)
  4. "RS Personal BS - 2024"   ← use for prior year balances (optional)

ON APP STARTUP, do the following:
  - Use SheetJS (xlsx library) to read and parse the Excel file at /data/2025_Rose_NEW_and_IMPROVED_Financials.xlsx
  - Extract all data programmatically from the sheets — do NOT hardcode any dollar values
  - Store parsed data in React state/context so all components read from it
  - If parsing fails, show a clear error state with a fallback upload prompt

PARSING STRATEGY:
  - Load the workbook with SheetJS: xlsx.readFile() or fetch + xlsx.read()
  - Convert sheet to array of arrays: xlsx.utils.sheet_to_json(sheet, {header:1})
  - Scan column B (index 1) for label text matches (use .includes() or .trim() === )
  - Once a matching row is found, extract the numeric values from the month columns
  - Month columns follow this pattern in the raw sheet:
      Col index 4  = Jan value
      Col index 7  = Feb value
      Col index 10 = Mar value
      Col index 13 = Apr value
      Col index 16 = May value
      Col index 19 = Jun value
      Col index 22 = Jul value
      Col index 25 = Aug value
      Col index 28 = Sep value
      Col index 31 = Oct value
      Col index 34 = Nov value
      Col index 37 = Dec value
      Col index 40 = TOTAL
  - Some cells may be null/undefined — treat as 0
  - Some months may have "Closed" text instead of a number — treat as 0
  - Build a dataService.js utility that exports a parsed financials object
  - The parsed object shape should be:

AI CHAT ASSISTANT:

BACKEND — /api/chat.js (Vercel Edge Function):
  - Accepts POST { messages: [], financialContext: {} }
  - Reads OPENROUTER_API_KEY from process.env (never exposed to client)
  - Calls OpenRouter: https://openrouter.ai/api/v1/chat/completions
  - Model: use OPENROUTER_MODEL env var, default to "anthropic/claude-sonnet-4-6"
  - Injects system prompt with the full financialContext JSON stringified
  - Returns streamed response
  - Handle CORS for Vercel

FRONTEND — ChatAssistant.jsx:
  - Floating cyan button bottom-right, opens slide-up dark panel
  - On open, passes current FinancialsContext data as financialContext to /api/chat
  - System prompt (injected server-side):
    "You are a financial assistant for Rosemary Schafer's personal and trust accounts.
     You have access to her complete 2025 financial data provided below as JSON.
     Answer questions clearly and concisely. Format all currency with $ and commas.
     When referencing specific months, use the month name. Explain LTC reimbursements
     as timing differences — money Will paid out of pocket being reimbursed from the
     trust, not actual losses. If asked about a specific account, reference its
     account number and type. Data: {financialContext}"
  - Support image uploads (base64, pass as vision message content)
  - Support document/CSV uploads (parse text, inject into user message)
  - Maintain last 10 messages in context
  - Show typing indicator while streaming
  - Render markdown in responses (use react-markdown)

IMPORTANT RULES FOR CLAUDE CODE:
1. Never hardcode any dollar amounts — all values must come from parsing the Excel file
2. dataService.js is the single source of truth for all parsing
3. If a cell is empty, null, or contains text like "Closed" — treat as 0
4. All chart components receive data as props from FinancialsContext
5. The Excel file stays in /data and is bundled with the app (no external fetch needed)
6. Make the app fully responsive for mobile
7. Add loading state while Excel is being parsed on first mount
8. Use Tailwind classes only — no custom CSS files
